**Requirement:** "Denetim logu: kim hangi yedekleme / Ansible komutunu ne zaman çalıştırdı kaydedilmeli"
**ROADMAP success criterion:** "After running a backup or Ansible playbook, the audit log *shows* which user triggered the action and when"

**Write side (WIRED):**
- `db.py`: `CREATE TABLE IF NOT EXISTS audit_log` ✓

- `routes/servers.py` line 440: `rear_configure_started` ✓
- `routes/servers.py` line 478: `backup_job_started` ✓
- `routes/ansible.py` line 347: `ansible_run_started` ✓
- All 4 sites: `session.get('username', 'anonymous')` + resource_id ✓

**Read side (MISSING):**
- `models/audit.py::get_audit_log()` defined at line 18
- **No route calls it** — no UI page, API endpoint, or template renders the audit log
- Users have no way to view audit history through the application

**Gap against success criterion:** The ROADMAP says "shows" which implies viewability. Data is captured but not accessible.
