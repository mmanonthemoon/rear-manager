CRITICAL url_for() updates: When endpoint names change from `'servers_list'` to `'servers.servers_list'`, update:
1. All `url_for()` calls in the route Python files
2. All `redirect(url_for(...))` calls
3. All `url_for()` calls in the Jinja2 templates (136 occurrences across 24 template files)

Blueprint endpoint naming convention: `url_for('blueprint_name.function_name')`. Examples:
- `url_for('servers_list')` -> `url_for('servers.servers_list')`
- `url_for('login')` -> `url_for('auth.login')`
- `url_for('dashboard')` -> `url_for('dashboard.dashboard')`
- `url_for('server_detail', sid=sid)` -> `url_for('servers.server_detail', sid=sid)`
- `url_for('job_detail', jid=jid)` -> `url_for('jobs.job_detail', jid=jid)`
- `url_for('ansible_dashboard')` -> `url_for('ansible.ansible_dashboard')`

Search EVERY template file for `url_for(` and update each call. This is the most error-prone step. Missing even one causes a BuildError at runtime.
